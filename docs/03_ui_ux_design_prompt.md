# UI/UX 설계 프롬프트 — 학폭 자문 앱 Flutter

> **용도**: Flutter 화면 구현 시 일관된 UX·카피·디자인 토큰을 유지하기 위한 가이드
> **수신자**: Claude Code (구현), 디자이너(협업), QA(검수)
> **컨텍스트**: 위기 상황(학폭 사건 직면)의 학부모·교사·학생이 사용. **차분하고 안심시키는 톤**이 절대 우선.

---

## 1. 디자인 원칙 (5가지)

### 1.1 차분함 (Calm)
- 위기 상황 사용자에게 채도 높은 색·강한 그림자·과한 애니메이션은 금기.
- 주요 색상은 **저채도 블루 + 차콜 그레이** 베이스. 강조는 절제된 포인트 컬러.

### 1.2 신뢰성 (Credible)
- 모든 답변에 출처 칩이 보이고, 면책 문구가 *눈에 보이는 위치에* 있어야 한다.
- 법령 인용은 인쇄물 같은 정돈된 타이포그래피 (한국어 본문 1.6 line-height).

### 1.3 중립성 (Neutral)
- 가해/피해 어느 쪽으로도 편들지 않는 톤.
- 아이콘 — ⚖️ 저울, 📘 책, 📞 전화 같이 **중립적 상징**만 사용. 경찰·법봉·감정적 일러스트 금지.

### 1.4 명료함 (Clarity)
- 한 화면 = 하나의 주된 행동 (질문 입력 / 조문 검색 / FAQ 선택 등).
- 본문 폰트 16sp 이상 (학부모 30~50대 가독성).

### 1.5 탈출구 (Escape)
- 어느 화면에서도 1탭으로 홈/이전/도움말로 갈 수 있어야 한다.
- 응급 신호 답변에는 117·1388·1393 등 외부 자원이 **상단**에 노출.

## 2. 디자인 토큰

### 2.1 색상 (Material 3 ColorScheme 시드)
```dart
// 시드 색
const seedColor = Color(0xFF2C5282);   // 차분한 진청

// Light
ColorScheme.fromSeed(seedColor: seedColor, brightness: Brightness.light)
  .copyWith(
    surface: const Color(0xFFFAFAFA),
    onSurface: const Color(0xFF1A1A1A),
    error: const Color(0xFFB23A3A),       // 면책·경고
  );

// Dark
ColorScheme.fromSeed(seedColor: seedColor, brightness: Brightness.dark);
```

### 2.2 면책 전용 색
- 면책 배너 배경: `errorContainer` (라이트: 옅은 살구, 다크: 짙은 회갈)
- 면책 텍스트: `onErrorContainer`
- **빨강을 너무 강하게 쓰지 말 것** — 위협이 아니라 안내.

### 2.3 타이포그래피 (Pretendard)
| 토큰 | 스펙 | 용도 |
|---|---|---|
| `displayLarge` | 28 / 700 / 1.3 | 온보딩 헤드라인 |
| `headlineMedium` | 22 / 700 / 1.4 | 화면 제목 |
| `titleLarge` | 18 / 600 / 1.5 | 섹션 헤더, 답변 강조구 |
| `bodyLarge` | 16 / 400 / 1.6 | 본문 (메시지 버블, 조문) |
| `bodyMedium` | 14 / 400 / 1.5 | 보조 설명 |
| `labelMedium` | 12 / 500 / 1.4 | 칩, 캡션, 면책 |

### 2.4 간격
- 기본 패딩: `16` (페이지), `12` (카드 안), `8` (요소 사이)
- 메시지 버블 간 간격: `12`

### 2.5 모서리 / 그림자
- BorderRadius: 16(카드), 24(버블), 999(칩)
- Elevation: 0~1만 사용. 그림자 대신 outline 1dp 권장.

### 2.6 모션
- 기본 이징: `Curves.easeOutCubic`, 200~300ms
- 페이지 전환: `MaterialPage` 기본
- 스트리밍 점멸: 1.2초 반복, opacity 0.4 ↔ 1.0

## 4. 화면별 명세

### S02 — 면책 동의 (Disclaimer)
**역할**: 변호사법 리스크 차단의 1차 관문.

**구성**:
- 상단: "잠깐, 시작 전에 꼭 알려드려요" (headlineMedium)
- 본문: 4가지 항목 (아이콘 + 1줄)
  1. 📘 본 앱은 **일반 법령·절차 정보 제공 서비스**입니다.
  2. ⚖️ **개별 사안에 대한 법률자문이 아닙니다.**
  3. 🔍 모든 답변은 **공식 법령·가이드북** 기반이며, **출처를 함께** 보여드려요.
  4. 📞 구체적 상담은 **변호사 또는 교육지원청**으로 문의해 주세요.
- 체크박스: "위 내용을 확인했으며 동의합니다." (필수)
- CTA: "동의하고 시작하기" (체크 안 하면 비활성)
- 하단 링크: 이용약관 / 개인정보처리방침

**카피 가이드**:
- ❌ "법률 상담", "변호사 대체"
- ✅ "정보 제공", "안내"

### S03 — 인트로 3페이지
| 페이지 | 헤드라인 | 서브 | CTA |
|---|---|---|---|
| 1 | "법령에 근거한 정보, 출처와 함께" | 학교폭력예방법·시행령·교육부 가이드북에서 직접 검색해 답변해 드려요. | 다음 |
| 2 | "어떤 질문이든 편하게" | "5호 처분이 뭐야?" 같은 일상어로 물어도 됩니다. 어려운 법률 용어는 풀어 설명해 드려요. | 다음 |
| 3 | "구체적 사안은 전문가에게" | 본 앱은 일반 정보 제공이며, 개별 사안 판단은 하지 않아요. 변호사·교육지원청 연계 정보도 함께 안내해 드립니다. | 시작하기 |

### S04~S05 — 홈/채팅
**레이아웃 (세로)**:
```
┌────────────────────────────────────┐
│ AppBar: "학폭 가이드"   [📞] [⚙]    │  ← 117·1388 빠른 연결, 설정
├────────────────────────────────────┤
│ [사용량 인디케이터]                 │
│ 남은 48회 / 오늘 50회               │
├────────────────────────────────────┤
│                                    │
│   메시지 리스트 (스크롤)             │
│                                    │
│   - 사용자 버블 (오른쪽, 진청)        │
│   - 어시스턴트 버블 (왼쪽, 차콜)      │
│     └ 인용 칩 1, 칩 2 ...            │
│     └ 면책 배너                     │
│                                    │
├────────────────────────────────────┤
│ [추천 질문 칩 가로 스크롤]           │
│ "5호 처분 뭐야?" "신고 절차"          │
├────────────────────────────────────┤
│ [입력바]  무엇을 알고 싶나요?    [↑] │
└────────────────────────────────────┘
```

**핵심 위젯**:

#### 1) 메시지 버블
- 사용자: 우측 정렬, `primaryContainer` 배경, `onPrimaryContainer` 텍스트
- 어시스턴트: 좌측 정렬, `surfaceContainerHigh` 배경, 응답 중에는 마지막 글자 옆에 `LoadingDots` (3개 점 점멸)
- 본문: `flutter_markdown` 렌더 (백엔드가 인용 표기 `[근거: ...]`를 텍스트에 포함하므로 인라인 강조)

#### 2) 인용 칩 (CitationChip)
- Material `Chip` 베이스, `outlined` 변형
- 라벨: `📘 학폭예방법 제17조`
- 탭 → 바텀시트 (S07) 호출
- 칩이 3개 초과면 "+N" 칩으로 축약

#### 3) 면책 배너 (DisclaimerBanner)
- 답변 버블 직하단, 폭은 버블과 동일
- `errorContainer` 배경, 12sp 텍스트
- 본문 (고정):
  > ⚖️ 본 답변은 일반 법령·절차 정보이며, 구체적 사안에 대한 법률자문이 아닙니다. 자세한 상담은 변호사 또는 관할 교육지원청에 문의하세요.
- 백엔드 답변에도 동일 문구가 들어 있을 수 있음 — 시각적 강조 차원에서 별도 위젯으로도 노출.

#### 4) 입력바 (InputBar)
- 멀티라인 (최대 4줄까지 자동 확장)
- 우측 전송 버튼: 입력 비어 있으면 비활성
- 송신 직전 `pii_detector`로 감지 → 의심 시 다이얼로그
- 송신 중에는 텍스트 필드 readonly, 우측 버튼은 "취소"로 변환 (스트림 cancel)

#### 5) 추천 질문 칩 (SuggestionChips)
- 첫 진입 시 5개 정적 제안:
  - "학폭위 조치 1~9호가 뭐야?"
  - "학교장 자체해결은 언제 가능해?"
  - "생기부에 얼마나 남아?"
  - "신고 절차 알려줘"
  - "학폭위 결과에 불복하려면?"
- 답변 후에는 백엔드가 추천 질문을 주지 않으므로 ⚠️ MVP에서는 정적.

#### 6) 사용량 인디케이터 (UsageIndicator)
- `남은 48회 / 일 50회` 작은 텍스트 + 진행바
- 남은 5회 이하: 색상이 `error`로 전환 + "이용권 보기" 링크
- 무료 플랜 한도 도달: 결제 안내 모달

### S07 — 조문 상세 바텀시트
- 상단: 법령명 (titleLarge), 조 번호 (labelMedium)
- 본문: `content` 그대로 (한국어 본문 1.6 line-height, monospace 금지)
- 하단: "원문 보기" → `source_url` 외부 브라우저 열기
- 시행일 표기: "시행일: 2024. 3. 1." 형식

### S08 — 절차 흐름도 (정적)
6단계 세로 카드 (탭 시 펼쳐지는 ExpansionTile):
1. 학폭 인지 → 신고 (117 또는 학교 전담기구)
2. 학교 전담기구 조사 (14일 이내 연장 가능)
3. 학교장 자체해결 여부 판단
4. 학폭위 소집 → 심의 → 조치 결정
5. 조치 이행 → 생기부 기재
6. 불복 (행정심판 → 행정소송)

각 단계 펼치면 관련 조문 인용 칩 1~2개 (S07로 연결).

### S09 — FAQ 퀵탭
`assets/faq/faq_seed.json`에서 로드. **LLM 호출하지 않음**.
스키마:
```json
[
  {
    "id": "faq_001",
    "question": "학폭위가 뭐야?",
    "answer_markdown": "...",
    "citations": [{"law": "학폭예방법 제12조", "url": "..."}]
  }
]
```
- 카드 리스트 → 탭 시 답변 화면 (메시지 버블과 동일 디자인). 면책 배너 동일.

### S10 — 설정·프로필
- 사용량 카드 (현재 플랜, 남은 횟수, 만료일)
- "이용권 구매" CTA (Phase 3)
- 이용약관 / 개인정보처리방침 / 오픈소스 라이선스
- 앱 버전, 빌드 번호
- 디버그 모드(개발 빌드만): API BASE_URL 토글, 헬스체크

### S12 — 피드백 다이얼로그
- 별점 1~5 (라디오 버튼 형태 ★)
- 옵션 코멘트 (멀티라인, 200자)
- 송신 → `POST /feedback`

### S13 — 신고 다이얼로그
- 신고 사유 라디오:
  - 출처가 잘못된 것 같아요
  - 사실과 다른 답변이에요
  - 개별 사안을 판단하는 표현이 있어요
  - 부적절한 내용이에요
  - 기타
- 코멘트 멀티라인 (선택)
- 송신 → `POST /feedback` with `rating=1, issue_type="report"`

## 5. UX writing (카피 통일)

### 5.1 톤
- **경어체** (~합니다, ~예요)
- "고객님" ❌ → "사용자" ✅ 또는 호칭 생략
- 위로보다 정보. "힘드시겠지만"보다 "관련 정보를 안내드릴게요".

### 5.2 용어 고정 (백엔드 시스템 프롬프트와 동일)
| 표기 | 풀어 쓸 때 |
|---|---|
| 학폭위 | 학교폭력대책심의위원회 |
| 학폭예방법 | 학교폭력예방 및 대책에 관한 법률 |
| 조치 | (처벌·처분 대신 사용) |
| 학교장 자체해결 | (학교장 종결 ❌) |
| 가해학생 / 피해학생 | (가해자/피해자 ❌ — 사실관계 미확정 표현 회피) |

### 5.3 핵심 카피 사전
| 상황 | 카피 |
|---|---|
| 첫 진입 | 무엇을 알고 싶으세요? |
| 사용량 0회 남음 | 오늘 무료 사용을 모두 사용했어요. 이용권을 확인해 보세요. |
| 네트워크 오류 | 인터넷 연결을 확인해 주세요. |
| 서버 오류 | 일시적인 오류가 발생했어요. 잠시 후 다시 시도해 주세요. |
| 응답 지연 | 답변을 준비하고 있어요... |
| 인용 없음 | 제공된 자료에서 정확한 정보를 찾지 못했어요. 변호사·교육지원청 상담을 권장해 드려요. |
| PII 감지 | 입력하신 내용에 개인정보로 보이는 부분이 있어요. **이름·학교명·연락처는 가급적 빼고** 질문해 주세요. |

### 5.4 금지 카피 (변호사법 / 과장 광고 방어)
- "승소", "패소", "무죄"
- "유리합니다", "확실해요", "예상됩니다"
- "법률 상담", "변호사 대체", "전문 자문"
- "100% 정확", "완벽한 답변"

## 6. 접근성 (a11y)

- 모든 인터랙티브 위젯에 `Semantics` 라벨 필수
- 최소 터치 타겟: 48dp
- 명도 대비: WCAG AA 이상 (자동 검사: `flutter_semantics_test`)
- 폰트 스케일링: 시스템 글자 크기 200%까지 깨지지 않게
- 색상만으로 정보 전달 금지 — 아이콘·텍스트 보조

## 7. 응급 신호 처리 (Critical Path)

답변 텍스트 또는 사용자 입력에 다음 키워드 감지 시 **상단 고정 배너** 노출:
- "자해", "자살", "죽고싶", "죽어버리"
- "위협받", "맞고있", "맞고 있"

배너 내용:
```
🚨 도움이 필요하면 지금 바로 연락해 주세요.
- 117 학교폭력 신고센터 (24시간)
- 1388 청소년상담전화 (24시간)
- 1393 자살예방상담
- 응급 시 112 / 119
```
- 탭 1회로 다이얼러 호출 (`tel:` 스킴)
- 백엔드 시스템 프롬프트도 동일 자원을 답변 본문에 포함하므로 **이중 안전망**.

## 8. 다크모드

- 시스템 테마 자동 추종 (수동 토글 X)
- 어두운 회청색 배경 + 부드러운 회백 텍스트
- 면책 배너는 다크에서도 가독성 검수 필수
- OLED 절약을 위해 순흑 `#000000` 대신 `#0E1218` 사용

## 9. 레이아웃 반응형

- 폰 세로: 기본
- 폰 가로: 메시지 리스트는 그대로, 입력바 하단 고정
- 태블릿: 좌측 350dp 사이드바(채팅 세션 목록 — Phase 3) + 우측 채팅
- 분기점: `MediaQuery.size.width >= 720`
