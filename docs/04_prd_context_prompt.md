# PRD 컨텍스트 프롬프트 — 타 AI 협업용 (Flutter 앱 관점)

> **용도**: ChatGPT, Gemini, Claude.ai, 또는 다른 Claude Code 세션에 학폭 자문 앱(Flutter 클라이언트) 작업을 위임할 때 컨텍스트를 한 번에 전달
> **사용법**: 아래 본문을 채팅창 맨 앞에 붙여넣은 뒤, 시나리오별 추가 프롬프트를 이어붙여 사용
> **현 시점 상태**: 백엔드(FastAPI + RAG)는 AWS 서울에 운영 중, **Flutter 클라이언트는 이제 시작**

---

## 베이스 프롬프트 (복사해서 사용)

```
나는 "학폭 가이드"라는 모바일 앱의 Flutter 프론트엔드를 개발 중이야. 백엔드는 이미 운영 중이고, 지금부터 모바일 앱을 만든다. 아래 맥락을 먼저 숙지하고 내가 이어서 요청하는 작업을 도와줘.

# 📌 프로젝트 한 줄 요약
한국의 학교폭력대책심의위원회(학폭위) 관련 법령·절차·조치를 공식 법령 기반 RAG로 안내하는 모바일 AI 자문 앱. 백엔드는 FastAPI + PostgreSQL(pgvector) + Claude Sonnet/Haiku로 이미 구축돼 AWS 서울에 운영 중이며, 이번 작업은 그 백엔드를 소비하는 Flutter 클라이언트.

# 🎯 배경
- 2024년 학폭예방법 개정으로 가해학생 조치가 생기부에 졸업 후 4년 보존, 2026 대입부터 감점.
- 변호사 상담은 1회 10~30만 원, 위기 초기 정보 탐색 단계엔 부담.
- 공식 가이드북은 PDF 281페이지, 인터넷 정보는 출처 불명·구법령 기준.
- 공신력 있는 법령 기반 AI가 그 간극을 메우는 기회.

# 👤 타겟
Primary:
- 학부모 (가해 측): 학폭위 소집 통보 받은 상황
- 학부모 (피해 측): 신고·보호조치·증거수집 필요

Secondary:
- 교사: 사안 인지 후 보고 절차
- 학생 본인 (무료 티어 중심)

# ✅ 앱 핵심 기능 (MVP)
1. **대화형 Q&A** — 자연어 질문 → 백엔드 SSE 스트리밍 답변 + 출처 칩
2. **조문 직접 조회** — 학폭예방법·시행령 조문 검색 (RAG 미경유)
3. **절차 흐름도** — 신고→조사→심의→조치→불복 시각화 (정적)
4. **FAQ 퀵탭** — 자주 묻는 10개, LLM 미호출 (자산 JSON)
5. **면책 고지** — 첫 실행 전체화면 + 답변마다 자동 배너
6. **답변 신고/평가** — Play 정책 필수
7. **사용량 인디케이터** — 무료 한도 가시화

Phase 3:
- Google Play 인앱결제 (7일/30일 이용권)
- 대화 이력 저장
- 푸시 알림 (법령 개정)
- iOS 빌드

# 🚫 절대 하지 않을 것 (변호사법 리스크)
- 개별 사안 결과 예측 ("5호 처분 받을 것")
- "승소 가능성", "유리한 전략" 같은 변호사 업무 표현
- 소장·의견서 등 법적 문서 직접 작성
- 클라이언트에서 LLM 직접 호출 (반드시 백엔드 경유)
- 결제 검증을 클라이언트만으로 처리 (백엔드 검증 필수)

# 🏗 Flutter 기술 스택 (확정)
- Flutter 3.x / Dart 3.x
- 상태관리: Riverpod (riverpod_annotation 코드 생성)
- 라우팅: go_router
- HTTP: dio
- SSE: 직접 구현 (또는 flutter_client_sse)
- JSON: freezed + json_serializable
- 저장소: flutter_secure_storage (Device ID), shared_preferences
- UI: Material 3, Pretendard 폰트, 다크모드 추종
- 결제 (Phase 3): in_app_purchase
- 관측: sentry_flutter
- 테스트: flutter_test + mocktail

# 🔌 백엔드 API 요약
- Base URL: https://school-advisor.sedoli.co.kr
- 인증: X-Device-ID 헤더 (UUID v4, 회원가입 없음)
- POST /api/v1/chat — SSE (event: text/citation/done/error)
- GET  /api/v1/laws?law=&article_no=
- POST /api/v1/feedback {conversation_id, rating, issue_type?, comment?}
- GET  /api/v1/user/status → {plan, questions_used, questions_limit, expires_at}
- Rate Limit: 분당 5회, 일일 50/150/500회 (free/7day/30day)

용어는 백엔드와 통일:

### 시나리오 C: SSE 디코더·Repository 작성

```
이번 작업: 학폭 자문 앱 백엔드의 /api/v1/chat SSE 응답을 받는 Dart 코드를 작성해줘.

조건:
- POST 요청, Content-Type: application/json, X-Device-ID 헤더
- 응답: text/event-stream, 프레임은 "event: ...\ndata: ...\n\n"
- 이벤트 종류: text / citation / done / error
- 디코더는 Stream<SseEvent>를 반환
- 청크 경계, 멀티바이트 경계, 누락 필드 모두 처리
- 취소 가능 (StreamSubscription.cancel)

출력:
1. SseEvent 데이터 클래스 (freezed)
2. SseClient.stream(...) 함수
3. ChatRepository.streamChat(...) — 도메인 모델로 매핑
4. 단위 테스트 — fake Stream<List<int>> 으로 다음 케이스
   - 정상 흐름 (text*N + citation + done)
   - 한 청크에 여러 프레임
   - 청크 경계에서 잘린 프레임
   - 한국어 multibyte 경계
   - error 이벤트
   - 스트림 취소
5. 에러 매핑 (DioException → AppException)

코드 컨벤션: very_good_analysis, 한국어 주석.
```

### 시나리오 G: 법무·약관 초안

```
이번 작업: 학폭 자문 앱의 이용약관과 개인정보처리방침 초안을 작성해줘.

이용약관 필수:
1. 서비스 목적 — "정보 제공 서비스", 법률자문 아님 명확히
2. 결제·환불 — 7일/30일 이용권, 7일 내 미사용 시 환불, 미성년자 결제 보호자 환불권
3. AI 생성 콘텐츠 고지 — 답변이 AI 생성, 오류 가능
4. 책임 제한 — 정확성 최선 노력, 개별 사안 책임 X
5. 금지 행위 — 봇·역공학·재배포·재판매
6. 미성년자 — 결제 시 성인 인증 / 무료 기능 무제한
7. 개인정보 처리 링크
8. 준거법·관할

개인정보처리방침 필수:
1. 수집 항목: Device ID(익명 UUID), 질문 내용(90일 보관), 결제 정보(Play가 보관, 우리는 토큰만)

3. 보유 기간: 90일 후 자동 익명화
4. 제3자 제공: Anthropic API(질문·답변 처리 목적), OpenAI(임베딩) — 약관 링크 포함
5. 사용자 권리: 삭제 요청 (Device ID 기준)
6. 보안 조치: HTTPS, RDS 암호화
7. 처리방침 변경 고지 절차

## 📝 사용 팁

### 1. 베이스 프롬프트는 한 번만
세션 시작 시 베이스 프롬프트를 1회 주입한 뒤, 같은 세션에서 추가 작업은 시나리오 프롬프트만 보내면 된다.

### 2. AI별 한계 인지
- **GPT-4o**: 톤이 일반 안내 쪽에 가까움. 한국 법률 도메인엔 약함. 검수 필수.
- **Gemini**: 안전 필터가 학폭 같은 민감 주제에서 과민 반응할 수 있음.
- **Claude (다른 세션)**: 일관성 좋음. 단, 백엔드 코드와 앱 코드 혼동하지 않도록 "이번엔 Flutter 단" 명시.

### 3. 일관성 유지
여러 AI가 다른 톤·용어를 쓰면 제품 일관성이 깨진다. 다음은 고정:
- "학폭위" / "학폭예방법" / "조치" / "학교장 자체해결"
- "가해학생·피해학생" (가해자/피해자 ❌)
- "정보 제공 서비스" (법률 상담 ❌)

### 4. 민감 산출물은 이중 검증
- 이용약관·개인정보처리방침 → 두 AI에 같은 작업 → 차이 비교 → 변호사 리뷰
- 마케팅 카피 → "변호사법", "과장 광고" 관점 별도 검토 패스

### 5. Flutter 코드는 실제 빌드해 검증
- AI가 만든 코드는 패키지 버전 가정이 다를 수 있음
- 받자마자 `flutter analyze` + `flutter test` 돌려보기
- riverpod_annotation 코드 생성은 `dart run build_runner build` 수행 필요
